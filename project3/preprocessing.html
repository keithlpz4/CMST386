CSS Preprocessors

Ways to implement preprocessing (client-side vs. server-side)

  • Server side – download and install node.js and the LESS package on the server afterwards utilizing the NPM you can compile the LESS file(s) into CSS file.
  • Client side- Link the LESS file in the head of html as it were a .css file but must also provide the client access to a .js script that allows the client to compile the LESS file into CSS.

Preprocessing mechanisms of LESS and SASS

SASS and LESS are dynamic preprocessors that complie into CSS code. These preprocessor allow developers to write less CSS code by utilizing various mechanisms like a programming language like JavaScript. These mechanisms include:

Variables – Like JavaScript, LESS utilize variables to store commonly used values. This allows the developer to call down the value in various CSS elements and change them all from a center location. This makes updating CSS scripts easier if several elements need to change.
Nested Variables – this feature allows developers to structure CSS elements in the same visual hierarchy as HTML to lessen the amount of code repletion in CSS and better organize code.
Mixins – Mixins – this a feature that allows developers to list several CSS elements as variables. This allows developers to pass these attributes into other CSS elements to look similar. For instance, if you have multiple article with the same padding or color attributes then you could insert the insert the mixin into those elements without having to re-write the same code again.
Operators – LESS also allows the use of arithmetical operations on variables. This will allow LESS to manipulate increase or decrease individual elements like color or text size without having to alter the called variable.
Functions – used to create more complex relationships between CSS properties. This feature allows the use of logical, string, math, and other misc. functions, making the creation of dynamic html code significantly easier.
